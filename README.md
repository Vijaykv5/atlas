# Atlas

Atlas is an Avalanche-powered memory archive. It lets people publish personal
memories with image and optional voice media, anchor the memory on Avalanche
Fuji, and mint the result as an ERC-721 NFT and show them to other people in the world.

Every Atlas memory has two durable references:

- an on-chain memory record created through the Atlas smart contract
- an NFT metadata URL served by the deployed app for embedded media and wallet previews

## What It Does

Atlas turns a memory into a verifiable NFT-backed record:

1. The creator writes a title, country, memory type, and note.
2. The creator embeds image media and can include voice media.
3. The app stores public NFT assets through its Next.js API.
4. The creator connects an EVM wallet and switches to Avalanche Fuji.
5. The wallet calls `createMemory` on the Atlas contract.
6. The contract stores the memory fields on-chain and mints an ERC-721 NFT to the creator.
7. The minted token's `tokenURI` points back to Atlas metadata for the embedded media.

Each memory ID is also the ERC-721 token ID, so a minted memory can be checked
directly on Snowtrace and held in the creator's wallet.

| Deploy script | [`blockchain/scripts/deploy-fuji.ts`](./blockchain/scripts/deploy-fuji.ts) |

The main contract call is:

```solidity
createMemory(
  string title,
  string country,
  string kind,
  string description,
  string imageCid
)
```

`imageCid` is the public metadata URL generated by Atlas, for example:

```text
https://atlas.vijaykv.xyz/api/memories/metadata/<mediaId>
```

When the transaction succeeds, the contract emits `MemoryCreated`, stores the
memory on-chain, and mints an `ATLAS` NFT to the caller.

## Embedded Media And Metadata

Atlas keeps the on-chain record compact while still making each NFT rich enough
for wallets, explorers and viewers:

- `POST /api/memories/assets` stores the image, optional voice media, and display fields before minting.
- `GET /api/memories/metadata/[mediaId]` returns ERC-721 JSON metadata.
- `GET /api/memories/media/[mediaId]` returns the embedded image media.
- `POST /api/memories` records the finalized wallet transaction after minting.
- `GET /api/memories` returns public memories for the globe and profile views.

The metadata response includes the memory title, description, country, type,
creator-facing attributes, and an `image` URL that resolves to the stored media.
The same metadata URL is saved on-chain as the token URI.

## Features

- Interactive globe for browsing public memories by country.
- Memory creation flow with title, country, type, description, image, and voice fields.
- Browser wallet connection for Avalanche Fuji.
- Network switching prompt for Fuji C-Chain.
- ERC-721 minting through the deployed Atlas contract.
- Snowtrace transaction links after publishing.
- Profile page for wallet-specific memories.
- Public NFT metadata and embedded media routes backed by Neon Postgres.

## Local Setup

Install dependencies:

```bash
bun install
```

Create the app environment file:

```bash
cp .env.example .env.local
```

Set the public contract and app values:

```env
NEXT_PUBLIC_ATLAS_CONTRACT_ADDRESS=0xeEfbf26250b01BBfEC3348a8094E3f6C1875044B
NEXT_PUBLIC_ATLAS_PUBLIC_APP_URL=http://localhost:3000
DB_URL=
```

Run the app:

```bash
bun run dev
```

Open http://localhost:3000.

## Blockchain Setup

Install contract dependencies:

```bash
cd blockchain
bun install
```

Create the contract environment file:

```bash
cp .env.example .env
```

Set:

```env
FUJI_RPC_URL=https://api.avax-test.network/ext/bc/C/rpc
DEPLOYER_PRIVATE_KEY=
```

Use a funded Fuji test wallet only. Never commit `.env`, never share private
keys, and never paste a seed phrase into the app.

Compile and test:

```bash
bun run check
```

Deploy to Avalanche Fuji:

```bash
bun run deploy:fuji
```

The deploy script prints the contract address, deployer address, transaction
hash, chain ID, and Snowtrace URL. Copy the contract address into
`NEXT_PUBLIC_ATLAS_CONTRACT_ADDRESS`.
